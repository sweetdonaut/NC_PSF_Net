# 策略 A（純真實底圖）設計筆記 — 已歸檔

> **本文件 status**：理論記錄，**已被 `docs/design_hybrid.md` 取代**。
>
> 純策略 A 的核心想法是「完全靠真實前後站影像當底圖，連持續類訓練 signal 也由底圖自然提供，省下 inpaint capacity」。深入分析後發現這個方向有結構性問題（強制四錨點機制湊不齊），最後演化成 Hybrid 策略 —— 用真實底圖取代合成底圖，但保留策略 B 的 inpaint 機制。
>
> 本文件保留作為設計決策紀錄，說明 hybrid 為何不是「折衷」而是「必然」。

---

## 1. 純策略 A 的核心想法

底圖 = 真實前站影像 + 真實「乾淨」後站影像（同位置配對）。

「乾淨後站」精確定義為「**沒有後站新增 defect**」，前站既有 defects 仍會自然延續到後站（這是真實生產的狀態）。

預期的 case 來源分配：

| Pattern | 預期來源 |
|---|---|
| `00→00` | 底圖隱含（背景） |
| `10→10` | **底圖隱含**（前站 T-only 自然延續） |
| `01→01` | **底圖隱含** |
| `11→11` | **底圖隱含** |
| `00→10` | inpaint（必須，因為「乾淨後站」排除） |
| `01→10` | inpaint（必須） |
| `00→01` | inpaint |
| `00→11` | inpaint |
| `11→10` | **底圖可能隱含**（製程選擇性移除） |
| `11→01` | **底圖可能隱含** |

**核心優勢**（相對 strategy B）：
- 真實前站 defect 外觀分布
- 真實跨站 noise 結構
- 持續類訓練 signal 來自真實事件，不是合成 PSF
- 省下 inpaint 持續類的 capacity

## 2. 為何純策略 A 不可行

### 2.1 強制四錨點機制湊不齊（最致命）

策略 B 的「後站 T-only 家族四錨點」（`00→10`, `01→10`, `10→10`, `11→10`）保證每個 patch 都有完整四種，逼模型學「只看 `prev_T`、忽略 `prev_R`」。

純策略 A 下：
- `00→10` 和 `01→10` 仍 inpaint → **每 patch 可控** ✓
- `10→10`：靠底圖 → **密度由真實前站 defect 數量決定，不可控**
- `11→10`：靠底圖 → **更稀疏**（製程選擇性移除是中等頻率事件）

→ 四錨點失去對等性，多數 patch 缺乏 `10→10` 或 `11→10` 樣本，模型在那些 patch 上可能發展 shortcut（例如「跨站對比後站獨有就標 1」）。

### 2.2 正樣本密度不可控

`00→10` 和 `01→10` 雖然 inpaint，但**它們不能落在底圖既有 defect 位置**（否則 GT 標錯）。

- 純策略 A 強調「不 inpaint 持續類」，但對 `00→10` 的 inpaint 仍要避撞底圖的 `10→10`/`11→10` 位置
- 需要前置 detector 找底圖 defect 位置 —— 引入新的失敗模式
- 不避撞則 GT 雜訊累積

### 2.3 GT 完整性風險（依賴篩選品質）

底圖中若有未檢出的真實 `00→10`（後站新 defect 被誤判為「乾淨」），其 GT 為 0（我們不知道它是 anomaly）。模型看到「視覺上是 anomaly + GT=0」→ 學到「這種 defect 不該標」→ recall 受傷。

這個風險策略 B 完全沒有（底圖完全乾淨），策略 A 因為使用真實後站，無論篩選多嚴格都有殘餘風險。

### 2.4 持續類訓練密度不可控

300 張影像中，真實前站 defect 數量假設每張 5 個 → 共 1500 個事件。切成 128×128 patch（每張約 16 個）→ 4800 patches → 平均每個 patch 只有 0.3 個持續類事件。

→ 大部分 patch 沒有任何持續類訓練 signal，模型在這些 patch 上失去「跨站對比」的核心訓練 case。

## 3. 演化路徑：純 A → Hybrid

問題拆解：
- 「真實底圖」的優勢應保留（真實 defect 外觀 + 真實 noise）
- 「強制錨點」的優勢應保留（可控訓練分布 + 無 shortcut）
- 兩者並不衝突 —— 可以同時拿到

**Hybrid = 真實底圖 + 沿用策略 B 的全部 inpaint 機制**

詳細設計見 `docs/design_hybrid.md`。

關鍵 trade-off：
- 失去純策略 A 的「省 inpaint capacity」優勢 —— 但這個優勢本來就是空談，因為錨點機制需要它
- 引入「inpaint 撞到底圖既有 defect」的小機率 GT 雜訊 —— 但比例可估算為 < 1%，可忽略
- 換取真實底圖的所有優勢
- 換取訓練分布可控的所有優勢

---

## 4. 留給未來的問題

若 hybrid 在實機驗證上仍有瓶頸（例如 recall 受真實底圖中未檢出 anomaly 影響），可考慮的方向：

1. **方案 2（threshold 避撞）**：inpaint 前粗略檢查 `prev_T` 該位置強度，超過 threshold 跳到下個隨機位置
2. **方案 3（detector 避撞）**：用前置 detector 標出底圖 defect mask，inpaint 嚴格避開
3. **方案 4（GT 矯正）**：inpaint 後重新計算正確 GT，需要知道底圖 defect 位置 + 強度
4. **方案 5（更嚴格篩選）**：強化「乾淨後站」篩選機制，可能透過多次篩選 + 多 inspector 投票
